列表套字典

Python 最常见的数据结构，一图看懂

只用列表

只能存名字：
"小明"
"小红"
只有一个维度的信息

只用字典

只能存一个人：
"name": "小明"
"age": 20
"city": "北京"
只能装一个对象

列表 + 字典

多个人 x 多个属性：
[ {人1}, {人2}, {人3} ]
完美！既多又全

数据结构全貌

外层是列表 [ ]，里面每个元素是一个字典 { }

怎么取数据？

users[0]
→
{"name": "小明", "age": 20, "city": "北京"}
先用索引，从列表里取出第 1 个字典
users[0]["name"]
→
"小明"
再用 key，从字典里取出具体的值
users[2]["city"]
→
"广州"
索引 2 = 第 3 个人（小李），key "city" = 城市
