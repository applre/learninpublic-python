15 # 2. 但如果参数已经存在于一个字典里呢？
引出实际问题：参数在字典中怎么传？
16 info = {"name": "小白", "age": 25, "job": "产品经理"}
info→变量名 = 赋值 "name"→key（字符串） "小白"→value（字符串） key 和函数参数名完全一致

18 # 笨办法：一个个取出来传
对比：笨办法 vs 聪明办法
19 make_card(info["name"], info["age"], info["job"])
info["name"]→用 key 取出 value 三个参数各取一次，很繁琐
20
21 # 聪明办法：用 ** 一次性"解包"
核心：** 解包运算符
22 make_card(**info) # 等价于 make_card(name="小白", age=25, job="产品经理")
**→解包运算符（把字典拆成 key=value） info→要解包的字典变量 等价于手动写三个 key=value 参数
23
24 # 三种写法，效果完全一样！
三种写法：直接传值 / info[key] 取值 / ** 解包，结果完全相同

30 # 3. ** 就是"把字典拆开，变成 key=value 的形式"
** 的本质定义
31 #
32 # 原始字典：{"name": "小白", "age": 25, "job": "产品经理"}
解包前：三个 key-value 打包在字典里
33 #
34 # **字典 → name="小白", age=25, job="产品经理"
解包后：每个 key=value 变成独立的具名参数
35 #
36 # 就像拆快递：
类比：字典 = 快递盒，** = 拆开
37 # 📦 {"name": "小白", "age": 25} ← 打包的字典
📦 字典 = 打包好的快递盒
38 # ** 拆开 → name="小白", age=25 ← 拆成一个个参数
** 拆开 → 每个 key=value 变成独立参数

49 # 5. 字典的 key 必须和函数参数名对得上
重要规则：key 必须与参数名一致
50 def add(a, b):
def 定义函数 add→函数名：相加 a, b→两个参数
51 return a + b
return→返回计算结果 a + b→两数相加
52
53 numbers = {"a": 10, "b": 20}
numbers→字典变量 "a": 10，"b": 20 key 与函数参数 a, b 完全对应
54 result = add(**numbers) # 等价于 add(a=10, b=20)
result→接收返回值的变量 **→解包 numbers→被解包的字典 等价于 add(a=10, b=20)
55 print(f"10 + 20 = {result}")
print 打印结果 输出：10 + 20 = 30

**dict → 解包
**info 把字典解包成 key=value 形式的具名参数，直接传给函数
字典存函数
